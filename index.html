03
Upload Your Files
Two methods — browser upload (easy) or Git CLI (recommended)
3.1
Prepare your files — rename the main file to index.html
GitHub Pages serves index.html as the homepage automatically. Rename your main file, then organize your files like this:
ai-trading-platform/
index.html ← rename ai_trading_platform.html to this
slippage-tracker.html ← keep this name (secondary page)
README.md ← auto-created by GitHub
3.2a
Method A — Browser Upload (No coding required)
Inside your new repository, click Add file → Upload files. Drag and drop both HTML files into the upload area. Add a commit message like "Add trading platform files" and click Commit changes.
github.com/nexaforgeai/ai-trading-platform/upload

index.html · slippage-tracker.html
Commit message
3.2b
Method B — Git CLI (Recommended for ongoing updates)
Open Terminal (Mac/Linux) or Git Bash (Windows). Run these commands one at a time from the folder where your HTML files live. This method makes future updates much faster.
bashTerminal — run in your project folder
# Step 1: Initialize git in your project folder
git init

# Step 2: Connect to your GitHub repo (replace with YOUR username)
git remote add origin https://github.com/nexaforgeai/ai-trading-platform.git

# Step 3: Stage all your files
git add .

# Step 4: Commit with a message
git commit -m "Initial commit: AI Trading Platform + Slippage Tracker"

# Step 5: Set branch to main and push
git branch -M main
git push -u origin main
i
Don't have Git installed? Download it free at https://git-scm.com/downloads — takes 2 minutes to install on Windows or Mac.

6.2
Add collaborators for private feedback
To give specific people write access or manage feedback: go to Settings → Collaborators → Add people. Anyone can view a public repo and use a public GitHub Pages URL without being a collaborator.
6.3
Use GitHub Issues to collect tester feedback
Click the Issues tab in your repo and enable it. Share this with testers: "Found a bug or have feedback? Open an issue at [repo URL]/issues". This gives you a clean, organized feedback pipeline.
07
Push Updates
How to update the live site after making changes
7.1
After editing your HTML files locally, push changes
Every time you change your files and push to the main branch, GitHub automatically redeploys — usually live within 60 seconds. Use these 3 commands every time.
bashRun these 3 commands after every update
# Stage all changed files
git add .

# Commit with a descriptive message
git commit -m "Fix: slippage chart now shows market vs limit split"

# Push to GitHub (triggers automatic redeployment)
git push
✓
GitHub Pages auto-deploys on every push. Check the Actions tab to watch it go live — typically 30–90 seconds.
7.2
Edit files directly in the browser (no Git needed)
For quick fixes: navigate to the file in your repo, click the pencil icon to edit, make your change, and click Commit changes. The site updates automatically — no terminal required.
!
Cache tip: After deploying, your browser may show the old version. Hard refresh with Ctrl+Shift+R (Windows) or Cmd+Shift+R (Mac), or open in incognito mode to see the latest version.
08
Custom Domain (Optional)
Replace the github.io URL with your own domain
8.1
Point your domain to GitHub Pages
If you own a domain (e.g., nexaforgeai.com), you can serve your GitHub Pages app from it. Two steps: add a CNAME file to your repo, then configure DNS at your registrar (Namecheap, GoDaddy, etc.).
fileCreate a file named exactly: CNAME (no extension)
trader.nexaforgeai.com
Then at your DNS provider, add a CNAME record:
dnsAdd this CNAME record at your domain registrar
Type:  CNAME
Name:  trader
Value: nexaforgeai.github.io
TTL:   3600
i
DNS propagation takes 5 minutes to 48 hours. HTTPS is automatic once GitHub detects your domain is pointing correctly — no SSL certificate setup needed.
